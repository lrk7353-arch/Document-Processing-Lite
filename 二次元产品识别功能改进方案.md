# 二次元周边产品识别功能改进方案

> 文档版本：v1.0
> 创建日期：2026-04-07
> 项目名称：文档智能治理中台 (Document Intelligent Process Agent for OpenClaw)

---

## 一、需求背景

### 1.1 现有系统能力
- **外贸发票/单证识别**：已完成 OCR + 大模型提取 + 合规校验
- **技术栈**：React前端 + Python后端 + LangGraph Agent + SSE实时通信
- **核心功能**：文件上传 → 字段提取 → 规则校验 → 结果展示

### 1.2 新增业务场景
将文档识别能力从**外贸发票**扩展至**二次元周边产品**，服务于：
> 二次元周边产品的管理、分析及录入
> 特点：SKU极多、IP与角色特征明显且深度绑定

### 1.3 功能定位
**新增功能，非取代**：保留现有外贸发票识别模块，新增二次元产品Excel识别模块，两者并行。

---

## 二、目标与范围

### 2.1 核心目标
对固定模板的Excel表格中的二次元周边产品行数据进行**自动识别提取**与**合规检查**，输出可供ERP系统导入的标准化数据。

### 2.2 输入/输出定义

| 类型 | 说明 |
|------|------|
| **输入** | 固定模板格式的Excel文件（.xlsx/.csv） |
| **输出** | 提取并校验后的Excel/JSON，包含各字段状态（成功/失败/待确认） |
| **ERP对接** | Excel导入模式（识别结果导出后手动导入ERP） |

## 四、功能模块设计

### 4.1 模块架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Excel 识别系统                            │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │  Excel Parser │→│ Field Extract │→│ Compliance   │      │
│  │  (模板解析)   │  │ (字段提取)    │  │ Check       │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
│         │                 │                  │              │
│         ↓                 ↓                  ↓              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │  Image Proc  │  │  IP/Char     │  │  Barcode     │      │
│  │  (图片校验)   │  │  Disambig    │  │  Validator   │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 子模块详细设计

#### 4.2.1 Excel Parser（模板解析）
- **职责**：读取Excel文件，识别表头与数据行
- **输入**：.xlsx / .csv 文件
- **输出**：结构化行数据列表
- **依赖**：openpyxl / pandas

#### 4.2.2 Field Extractor（字段提取）
- **职责**：按模板列名映射提取各字段值
- **策略**：
  - 直接匹配：列名与字段名一一对应
  - 模糊匹配：处理列名微小差异（如"商品名称" vs "Product Name"）
  - 空白填充：缺失字段标记为 `NULL`

#### 4.2.3 Image Compliance Check（图片合规校验）
- **职责**：验证图片与商品描述的一致性
- **检查项**：
  1. 图片与商品名称（CN/EN）匹配度
  2. 图片与L1/L2分类匹配度
  3. 图片与IP/Character匹配度
- **实现方式**：多模态大模型（GPT-4V或同等能力模型）进行图文一致性判断
- **输出**：匹配度评分 + 通过/不通过/待确认

#### 4.2.4 IP/Character Disambiguation（IP角色消歧）
- **职责**：解决同一角色名可能属于不同IP的问题
- **策略**：
  - **知识库优先**：维护IP-Character关联知识库
  - **上下文推断**：无明确匹配时，根据同批次数据中IP共现情况推断
  - **不强制**：无法消歧时标记 `?` 待人工确认
- **示例**：
  ```
  角色"雷电将军" → 知识库匹配IP=原神 ✓
  角色"亚丝娜"   → 知识库匹配IP=刀剑神域 ✓
  角色"不知火舞" → 知识库无匹配，标记待确认
  ```

#### 4.2.5 Barcode Validator（条形码校验）
- **职责**：校验Barcode格式与校验位
- **规则**：
  - 固定13位数字
  - 校验位符合EAN-13算法
- **输出**：有效/无效/格式错误

#### 4.2.6 Price/Quantity Compliance（价格数量校验）
- **职责**：检查价格与数量字段的逻辑一致性
- **检查项**：
  1. `Cost_CN` > 0
  2. `SRP_CN` > `Cost_CN`
  3. `DISC%` = `Cost_CN` / `SRP_CN`（允许±5%误差）
  4. `TH/MY/SG/IND` 各零售价 > 0
  5. `Cabinet` 为正整数
  6. 跨境价格比率合理性（与人民币零售价的汇率偏差检测）

#### 4.2.7 Content Position Check（填写位置校验）
- **职责**：检测必填字段是否正确填写在对应位置
- **规则**：
  - `Barcode` 不得为空
  - `Product_Name_CN` 不得为空
  - `IP` 不得为空
  - `Character` 不得为空
  - `Cost_CN` 不得为空
  - `SRP_CN` 不得为空

## 五、合规检查汇总

| 检查类型 | 检查内容 | 判定结果 |
|----------|----------|----------|
| **字段填写位置** | 必填字段非空 | ✓通过 / ✗缺失 |
| **Barcode正确性** | 13位EAN-13格式与校验位 | ✓有效 / ✗无效 |
| **内容合规性** | 文本字段无异常字符/敏感词 | ✓通过 / ✗违规 |
| **价格数量合规** | 数值范围、逻辑关系校验 | ✓通过 / ✗异常 |
| **图片合规** | 图片与商品描述一致性 | ✓匹配 / ✗不匹配 / ?待确认 |
| **IP/Character消歧** | 角色-IP关联度评估 | ✓确认 / ?待确认 |

## 六、数据流设计

### 6.1 处理流程

```
[Excel文件上传]
      ↓
[Excel解析] → 识别表头与数据行
      ↓
[字段提取] → 按模板列名映射
      ↓
┌─────────────────────────────────────┐
│  并行执行以下校验：                   │
│  ├── Barcode校验                      │
│  ├── 价格数量校验                      │
│  ├── 图片一致性校验（调用多模态模型）    │
│  └── IP/Character消歧                  │
└─────────────────────────────────────┘
      ↓
[结果聚合] → 生成报告
      ↓
[导出] → 带校验状态的Excel/JSON
```

### 6.2 错误处理策略

| 错误类型 | 处理方式 |
|----------|----------|
| 读取失败 | 标记行级别错误，跳过继续 |
| 字段缺失 | 标记为 `NULL`，不影响其他行 |
| Barcode无效 | 标记错误，不阻断整批处理 |
| 图片调用失败 | 降级为文字描述匹配，标记待确认 |
| IP消歧失败 | 标记 `?`，不阻断后续流程 |

## 七、知识库设计（IP/Character消歧）

### 7.1 核心数据表

```sql
CREATE TABLE ip_character_knowledge (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    ip_name     VARCHAR(100) NOT NULL,      -- IP名称
    character   VARCHAR(100) NOT NULL,       -- 角色名称
    confidence  DECIMAL(3,2) DEFAULT 1.0,    -- 关联置信度
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE ip_alias (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    ip_name     VARCHAR(100) NOT NULL,      -- 标准IP名
    alias       VARCHAR(100) NOT NULL,      -- 别名
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 7.2 消歧算法

```
输入：character_name
输出：likely_ip_list / UNCERTAIN

1. 精确匹配知识库 → 返回匹配IP
2. 别名表展开名称后重试
3. 模糊匹配（如拼音近似）→ 返回高置信度候选
4. 无匹配 → 标记 UNCERTAIN
```
